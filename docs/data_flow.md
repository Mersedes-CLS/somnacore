# Data Flow: от повторения до записи

## Схема потока данных

```
ФИЗИКА → ЖЕЛЕЗО → ESP32 → СЕРВЕР → ПОЛЬЗОВАТЕЛЬ

┌─────────────────────────────────────────────────────────────────┐
│ 1. ИДЕНТИФИКАЦИЯ                                                │
│                                                                 │
│  [Ntag213 браслет] ──tap──► [PN532 ридер]                      │
│                                    │                            │
│                               читает UID                        │
│                                    │                            │
│                             [ESP32] ──HTTP──► /session/start    │
│                                    │         {uid, machine_id}  │
│                             [OLED]: "Привет, Иван!"             │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 2. ОПРЕДЕЛЕНИЕ ВЕСА (однократно при старте подхода)             │
│                                                                 │
│  [VL53L0X №1]  ←── смотрит на штифт в стеке                   │
│       │                                                         │
│  расстояние D → сравниваем с таблицей калибровки               │
│       │         D=142мм → 30кг                                  │
│       │         D=98мм  → 50кг                                  │
│       ▼                                                         │
│  weight = 30кг  (фиксируем, пока штифт не двигается)           │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 3. ПОДСЧЁТ ПОВТОРЕНИЙ (непрерывно во время подхода)            │
│                                                                 │
│  [VL53L0X №2]  ←── смотрит на движение стека (сверху/снизу)   │
│       │                                                         │
│  stream: 820, 780, 650, 480, 350, 480, 650, 780, 820 мм        │
│       │                                                         │
│  rep_detector:                                                  │
│    ↓ вниз (к датчику) = concentric фаза                        │
│    ↑ вверх (от датчика) = eccentric фаза                       │
│    полный цикл вниз→вверх = 1 REP ✓                            │
│       │                                                         │
│  [OLED]: "30кг  Rep: 5"                                        │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 4. ОПРЕДЕЛЕНИЕ КОНЦА ПОДХОДА                                    │
│                                                                 │
│  триггер: стек не двигается > 3 сек                            │
│       │                                                         │
│  SET COMPLETE:                                                  │
│  { weight: 30, reps: 8, rom: 470мм, duration: 42сек }          │
│       │                                                         │
│  [OLED]: "Подход: 30кг x 8  Отдых..."                         │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 5. ОТПРАВКА НА СЕРВЕР                                           │
│                                                                 │
│  ESP32 ──WiFi──► POST /session/set                             │
│  {                                                              │
│    session_id: "abc123",                                        │
│    set_number: 2,                                               │
│    weight_kg: 30,                                               │
│    reps: 8,                                                     │
│    rom_mm: 470,                                                 │
│    duration_sec: 42,                                            │
│    timestamp: 1741420800                                        │
│  }                                                              │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 6. БАЗА ДАННЫХ (сервер)                                         │
│                                                                 │
│  users      { id, telegram_id, nfc_uid, name }                 │
│  sessions   { id, user_id, machine_id, date, total_volume }    │
│  sets       { id, session_id, weight, reps, rom, duration }    │
└─────────────────────────────────────────────────────────────────┘
                    │
                    ▼
┌─────────────────────────────────────────────────────────────────┐
│ 7. TELEGRAM MINI APP                                            │
│                                                                 │
│  [Сегодня — Блочная тяга]                                      │
│  Подход 1: 30кг × 8  ROM 470мм                                 │
│  Подход 2: 30кг × 7  ROM 465мм                                 │
│  Подход 3: 35кг × 5  ROM 420мм                                 │
│  Объём: 1765кг                                                  │
│                                                                 │
│  [История за 30 дней] [График прогресса]                       │
└─────────────────────────────────────────────────────────────────┘
```

## Железо

| Компонент | Роль | Интерфейс |
|-----------|------|-----------|
| VL53L0X №1 | Определение веса (позиция штифта) | I2C |
| VL53L0X №2 | Счёт повторений (движение стека) | I2C (другой адрес или XSHUT мультиплексинг) |
| PN532 | Чтение NFC браслета | I2C |
| Ntag213 | Браслет пользователя (UID) | — |
| OLED 128x64 | Локальная индикация на тренажёре | I2C |
| ESP32 | Центральный контроллер + WiFi | — |

## Состояния FSM (ESP32)

```
IDLE
  │  tap браслета
  ▼
IDENTIFYING  ──► /session/start → session_id получен
  │
  ▼
WAITING_FOR_SET  (показываем вес, ждём движения)
  │  стек начал двигаться
  ▼
IN_SET  (считаем репы, меряем ROM)
  │  стек остановился > 3 сек
  ▼
SET_COMPLETE  ──► POST /session/set
  │
  └──► WAITING_FOR_SET  (следующий подход)
  │
  │  tap браслета или timeout 10 мин
  ▼
SESSION_END  ──► POST /session/end
  │
  ▼
IDLE
```

## REST API (сервер)

```
POST /session/start        { nfc_uid, machine_id }        → { session_id, user_name }
POST /session/set          { session_id, set_number,      → { ok }
                             weight_kg, reps, rom_mm,
                             duration_sec, timestamp }
POST /session/end          { session_id }                  → { summary }
GET  /user/:telegram_id/history                            → [ sessions ]
```
