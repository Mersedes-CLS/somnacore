Синхронизация документации: $ARGUMENTS

## Что делать

1. **Прочитай текущие доки**: ARCHITECTURE.md, CLAUDE.md, BACKEND.md
2. **Просканируй код** — сравни с документацией:

### ARCHITECTURE.md
- Дерево файлов актуально? Новые/удалённые файлы?
- Диаграмма системы соответствует реальности?
- Таблица модулей firmware полная?
- API endpoints все перечислены?
- Схема БД актуальна?
- Константы config.h совпадают?

### CLAUDE.md
- Инструкции для Claude актуальны?
- Нет ли новых ограничений/правил?

### BACKEND.md
- Все endpoints перечислены?
- Форматы запросов/ответов актуальны?

3. **Покажи diff** — что устарело, что нужно обновить
4. **Жди ОК**
5. **Обнови файлы**

## Правила
- Только факты из кода, не придумывай
- Сохраняй стиль существующих документов
- Если изменений много — покажи план перед правками
